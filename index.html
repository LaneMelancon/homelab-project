Config #1 — Privacy VPN

Purpose: Hide all internet traffic from ISP. Websites see Hetzner IP, not your home IP.

Use when: Banking, private browsing, any session where you want IP anonymity.

ini — Device Config 
[Interface]
PrivateKey = <DEVICE_PRIVATE_KEY>
Address    = 10.0.0.2/24
DNS        = 10.0.0.1  # Unbound on VPS

[Peer]
PublicKey        = <VPS_PUBLIC_KEY>
Endpoint         = <HETZNER_IP>:51820
AllowedIPs       = 0.0.0.0/0  # All traffic → VPS
PersistentKeepalive = 25
ini — VPS Server Config 
[Interface]
Address    = 10.0.0.1/24
PrivateKey = <VPS_PRIVATE_KEY>
ListenPort = 51820
PostUp     = iptables -A FORWARD -i wg0 -j ACCEPT; iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
PostDown   = iptables -D FORWARD -i wg0 -j ACCEPT; iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE

[Peer]
PublicKey  = <DEVICE_PUBLIC_KEY>
AllowedIPs = 10.0.0.2/32

Config #2 — Remote Home Access

Purpose: Tunnel into your home network from anywhere. Access Plex and NAS as if sitting at home.

Use when: Traveling and want to stream Plex or access NAS files.

ini — Device Config 
[Interface]
PrivateKey = <DEVICE_PRIVATE_KEY>
Address    = 10.1.0.2/24
DNS        = 10.1.0.1

[Peer]
PublicKey        = <HOME_SERVER_PUBLIC_KEY>
Endpoint         = yourname.duckdns.org:51821
AllowedIPs       = 10.1.0.0/24, 192.168.1.0/24  # Home only
PersistentKeepalive = 25
ini — Home Server Config 
[Interface]
Address    = 10.1.0.1/24
PrivateKey = <HOME_PRIVATE_KEY>
ListenPort = 51821

[Peer]
PublicKey  = <DEVICE_PUBLIC_KEY>
AllowedIPs = 10.1.0.2/32

Router: Forward UDP 51821 to your home server's local IP.

VPS Setup — Hetzner Debian 12

bash — VPS Setup 
# Install WireGuard and Unbound
apt update && apt install wireguard unbound -y

# Generate server keys
wg genkey | tee /etc/wireguard/privatekey | wg pubkey > /etc/wireguard/publickey

# Enable IP forwarding
echo "net.ipv4.ip_forward=1" >> /etc/sysctl.conf && sysctl -p

# Firewall — strict allow list
ufw default deny incoming
ufw default allow outgoing
ufw allow 2222/tcp   # SSH non-standard port
ufw allow 51820/udp  # WireGuard Config #1
ufw enable

# Start WireGuard
systemctl enable wg-quick@wg0
systemctl start wg-quick@wg0

Home Server Setup

bash — Home Server Setup 
apt install wireguard -y

wg genkey | tee /etc/wireguard/privatekey | wg pubkey > /etc/wireguard/publickey

echo "net.ipv4.ip_forward=1" >> /etc/sysctl.conf && sysctl -p

ufw default deny incoming
ufw default allow outgoing
ufw allow 2223/tcp   # SSH (different from VPS)
ufw allow 51821/udp  # WireGuard Config #2
ufw enable

systemctl enable wg-quick@wg0
systemctl start wg-quick@wg0

# DuckDNS — runs every 5 min
echo "*/5 * * * * root curl -s 'https://www.duckdns.org/update?domains=yourname&token=YOUR_TOKEN&ip=' > /dev/null" >> /etc/cron.d/duckdns

Security Hardening

Critical
Key-only SSH Authentication
Disable password auth entirely. No brute force possible without your private key.
Critical
Strict UFW Firewall
Default deny all incoming. Allow only SSH (non-standard port) and WireGuard UDP port.
Critical
Non-Standard SSH Port
Move SSH off port 22. Eliminates 99% of automated scanner traffic immediately.
High
Fail2Ban
Automatically bans IPs that repeatedly fail authentication. Handles constant background noise silently.
High
Unattended Security Updates
Security patches apply automatically. No manual SSH required for routine CVEs.
High
Per-Device WireGuard Keys
One key pair per device. Compromised device = remove one peer. Everything else stays secure.
Medium
Disable IPv6
Eliminates a common leak vector. You don't need it — remove the attack surface.
Medium
Rotate Keys Every 6–12 Months
Limits exposure window if a key is ever silently compromised.

Component Rationale

Intel N100 — ASRock N100DC-ITX
~6W idle power draw running 24/7. Built-in Intel QuickSync for Plex hardware transcoding. CPU is soldered — no separate purchase needed. 2.5GbE port built-in; faster than the 1Gbps router port, which becomes the practical bottleneck at ~115 MB/s real-world.
Jonsbo N2
222 × 222 × 224mm compact cube — 11L total volume. 5 hot-swap 3.5″ bays plus a 2.5″ SSD tray. Hot-swap means a failed drive can be replaced without powering down. Requires SFX PSU (150mm max).
16GB Single Stick DDR4 SODIMM
One stick leaves the second slot open. Add another 16GB for ~$30 when needed. Board supports 32GB maximum.
2× Seagate IronWolf 8TB
NAS-rated drives built for 24/7 operation. 1M hours MTBF. 3-year warranty. Starting with 2 drives leaves 3 bays completely open for future expansion.

Connection Speeds

The N100DC-ITX has a built-in 2.5GbE port. Your router's 1Gbps ports are the bottleneck — still delivering ~115 MB/s real-world local transfer speed. More than sufficient for multiple simultaneous 4K Remux streams.

For remote access, your 1Gbps symmetric fiber upload means a 4K Remux stream at 80Mbps uses only 8% of available upload bandwidth. No constraints on remote quality.

Media Library & Plex

How Plex Works

Media files on NAS storage
        ↓
Plex Media Server  ·  runs 24/7 on NAS
Scrapes metadata — posters, cast, ratings
Organizes into Netflix-style library
        ↓
Plex app on Apple TV / iPhone / iPad
Requests content → server streams it

Direct Play vs Transcoding

Direct Play — Always preferred
Client natively plays the file format. Plex sends it as-is. Zero CPU usage on your server. Apple TV 4K direct plays the vast majority of common formats.
Transcoding — Rare fallback
Client can't handle the format. Plex converts on the fly. Intel N100's QuickSync hardware transcoding handles this efficiently when needed.

Quality Comparison

Format Bitrate File Size
4K UHD Remux 50–80 Mbps 50–80 GB
4K HDR x265 15–25 Mbps 15–25 GB
1080p Remux variable 20–40 GB
1080p x265 4–15 Mbps 4–15 GB
Netflix 4K 15–25 Mbps streamed

Your 4K Remux runs at 3–4× the bitrate Netflix delivers. Lossless TrueHD Atmos audio vs Netflix's Dolby Digital Plus. More retained detail in fast motion, dark scenes, and film grain.

Recommended Storage Format

  • Daily standard: 4K HDR x265 — visually indistinguishable from Remux, fraction of the file size
  • Favorites / reference: 4K UHD Remux — definitive quality for films that matter
  • 1080p content: x265 encoded — half the size of x264 with identical quality

Audio & Video Chain

Signal Chain

NAS
TrueHD Atmos source file
│

│ HDMI eARC
Sonos Arc
Decodes DD+ Atmos — Atmos logo appears ✓
│ Wireless
Sonos Sub Gen 2
Low frequency extension

Apple TV cannot pass raw TrueHD over eARC. It decodes TrueHD Atmos then re-encodes to Dolby Digital Plus with Atmos before sending to the Sonos Arc. The Atmos logo you see is real — full spatial audio processing is active.

The distinction vs Netflix: Your Remux starts from a higher bitrate lossless master. Even after Apple TV's transcode to DD+ Atmos, the result retains more audio information than Netflix's encode — resulting in noticeably better DD+ Atmos output.

Why Apple TV Stays Home

tvOS restricts persistent background VPN connections. Running WireGuard continuously on Apple TV isn't natively supported. Keeping Apple TV at home means it always streams locally at full quality with zero configuration — the ideal setup.

Why Not Tailscale

Tailscale is excellent but its coordination server isn't self-hosted unless you also run Headscale. For two devices and two configs, plain WireGuard is simpler and fully self-contained. Tailscale adds complexity that isn't needed here.

Why Not Plex Built-in Remote Access

Traffic routes through Plex's relay servers, and your home IP is either exposed or Plex's infrastructure is in the trust chain. WireGuard gives direct device-to-home connection with no intermediary involved.

Home Server Exposure

The home server has exactly one UDP port open (51821) on the router. WireGuard's cryptokey routing means this port is completely silent to anyone without your private keys. Scanners receive no response and move on. It's effectively invisible to the public internet.

Local LLM (Future)

The N100 build can run local language models today without hardware changes. CPU inference is slower than GPU but fully functional for personal use.

Model Size vs RAM

Model RAM Needed Capability
Llama 3.2 3B ~4 GB Basic, fast
Llama 3.1 8B ~8 GB Good all-rounder
Llama 3.1 8B Q4 ~5 GB Near-same quality, smaller
Mistral 7B ~8 GB Excellent for size
Llama 3.1 70B Q4 ~24 GB Very capable

With 16GB RAM and the N100: runs 7–8B models at ~10–20 tokens/second on CPU — usable for personal chat. Upgrade to 32GB to run quantized 70B models.

Software Stack

bash — Ollama Setup 
# Install Ollama
curl -fsSL https://ollama.com/install.sh | sh

# Pull and run a model
ollama pull llama3.1:8b
ollama run llama3.1:8b

Open WebUI provides a ChatGPT-style browser interface connected to Ollama. Accessible from any device on your home network or through the WireGuard tunnel remotely.

GPU Addition

The Jonsbo N2 accepts low profile GPUs up to 197mm only — a significant constraint. Better long-term path: keep the N2 for NAS/Plex/WireGuard (QuickSync handles transcoding), and add a separate small form factor PC with a full-size GPU for AI workloads.
